# SIKEY Roadmap (High-Level)

This is a living document. Everything here is subject to change as the community evolves.

## Phase 0 – Vision & Alignment (Now)

- Publish Manifesto v1 (`/docs/manifesto.md`)
- Publish Whitepaper v1 (`/docs/whitepaper.md`)
- Establish core principles, threat model, and non-negotiables

## Phase 1 – Protocol Design

- Define identity model and key hierarchy
- Choose / define DID strategy
- Design credential & attestation model
- Choose cryptographic primitives and ZK stack
- Publish initial SIKEY protocol specs in `/specs/`

## Phase 2 – Reference Implementation

- Build minimal SIKEY wallet / identity client
- Build basic verifier demo (web / CLI)
- Integrate at least one DID method
- Implement first ZK proof flow (e.g. age proof)

## Phase 3 – Ecosystem & Governance

- Formalise governance model
- Explore SIK token utility & design (if community agrees)
- Work with early adopters (apps, communities, orgs)
- Harden security, audits, and documentation

## Phase 4 – Hardening & Adoption

- Independent security audits
- Performance tuning and UX improvements
- Education, docs, and integration guides
- Push for adoption in real communities and apps
